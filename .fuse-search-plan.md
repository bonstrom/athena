# Fuse.js Fuzzy Search Plan

**Status:** Pending

## Goal

Replace manual `.filter()` substring matching in `GlobalSearch.tsx` with `fuse.js` fuzzy search, which handles:
- Typos ("cacy" → matches "cachy")
- Multi-word queries ("cachy linux" → matches content containing both words anywhere)
- Approximate matches (configurable threshold)

## Steps

### 1. Install dependency

```
npm install fuse.js
```

Version 7.3.0, ~11KB gzipped, no sub-dependencies.

### 2. Refactor `src/components/GlobalSearch.tsx`

**Imports:** Add `import Fuse from 'fuse.js';`

**Fuse configs:**

| Mode | Data source | Fuse keys | Key options |
|------|-----------|-----------|-------------|
| Topics | `athenaDb.topics.toCollection().filter(t => !t.isDeleted).toArray()` | `['name']` | threshold 0.4, includeScore, minMatchCharLength 2 |
| Messages | `athenaDb.messages.toCollection().filter(m => !m.isDeleted).toArray()` | `['content']` | threshold 0.4, includeScore, minMatchCharLength 2, ignoreLocation true |

**performSearch refactor:**
- Replace two-pass topic search (prefix indexed + substring scan) with single `fuse.search(query)`
- Replace message collection scan with `fuse.search(query)`
- Sort results by Fuse score (lower = better, more relevant first)
- Snippet generation: try `result.matches?.[0]?.indices[0]` for match position, fall back to `indexOf` for exact substring, fall back to first 60 chars
- Limit to 20 results as before
- Remove: prefix/substring split, `.where('isDeleted').equals(0)`, manual `.toLowerCase().includes()` logic

### 3. Update `src/components/GlobalSearch.test.tsx`
