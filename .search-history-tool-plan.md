# LLM Tool: `search_history` — Fuse.js Fuzzy Search for the LLM

**Status:** Planned — not implementing today

## Goal

Give the LLM a tool to fuzzy-search the user's conversation history (topics and messages) using the same fuse.js pipeline built into `GlobalSearch.tsx`. The LLM can call this to find relevant context from past conversations without needing the user to manually search.

## Background

The app currently has 4 LLM tools defined in `src/services/llmService.ts:69-145`:
- `update_scratchpad` — persist memory across turns
- `read_messages` — fetch specific messages by ID
- `list_messages` — directory of all messages (IDs + 150-char snippets, no search)
- `ask_user` — ask the user a follow-up question

These are:
- **Assembled** in `src/store/ChatStore.ts:989-999`
- **Executed** via `onExecuteTool` callback in `src/store/ChatStore.ts:878-987`
- **Gated** by auth store toggles (`messageRetrievalEnabled`, `askUserEnabled`)
- **Orchestrated** by the tool loop in `src/services/llmService.ts:1216-1377` (max 10 iterations)

The proposed `search_history` tool fills the gap between `list_messages` (firehose of everything) and `read_messages` (exact ID lookup) — providing targeted, relevance-ranked search.

## Execution

In `src/store/ChatStore.ts`, within the `onExecuteTool` callback (around line 878), add a handler:

```ts
if (toolName === 'search_history') {
  const { query, mode = 'both' } = argsJson as { query: string; mode?: string };

  const results: string[] = [];

  if (mode === 'both' || mode === 'topics') {
    const topics = await athenaDb.topics
      .toCollection()
      .filter((t) => !t.isDeleted)
      .toArray();
    const fuse = new Fuse(topics, {
      keys: ['name'],
      threshold: 0.4,
      includeScore: true,
      minMatchCharLength: 2,
    });
    const topicResults = fuse.search(query).slice(0, 5);
    if (topicResults.length > 0) {
      results.push('## Topic matches');
      topicResults.forEach((r) => {
        results.push(`- [${r.item.name}] (topic ID: ${r.item.id})`);
      });
    }
  }

  if (mode === 'both' || mode === 'messages') {
    const messages = await athenaDb.messages
      .toCollection()
      .filter((m) => !m.isDeleted)
      .toArray();
    const fuse = new Fuse(messages, {
      keys: ['content'],
      threshold: 0.4,
      includeScore: true,
      includeMatches: true,
      minMatchCharLength: 2,
      ignoreLocation: true,
    });
    const messageResults = fuse.search(query).slice(0, 5);
    if (messageResults.length > 0) {
      // Fetch parent topic names
      const topicIds = Array.from(new Set(messageResults.map((r) => r.item.topicId)));
      const topics = await athenaDb.topics.bulkGet(topicIds);
      const topicNames = new Map(topics.filter(Boolean).map((t) => [t!.id, t!.name]));

      results.push('## Message matches');
      messageResults.forEach((r) => {
        const topicName = topicNames.get(r.item.topicId) || 'unknown topic';
        const snippet = r.item.content.substring(0, 300).replace(/\n/g, ' ');
        results.push(`- [${topicName}] ${snippet}${r.item.content.length > 300 ? '...' : ''}`);
      });
    }
  }

  return results.length > 0 ? results.join('\n') : 'No matching history found.';
}
```

**Result caps:**
- Max 5 topic matches
- Max 5 message matches (content trimmed to 300 chars each)
- Results formatted as markdown for easy LLM consumption

**Rate limiting:** The existing tool loop limit (10 iterations) applies automatically. No additional rate limiting needed unless abuse is observed.

## Risk Mitigation

- **Privacy:** Tool gated behind explicit user toggle (off by default)
- **Token blowout:** Results capped at 5 per mode, content trimmed to 300 chars
- **Tool loops:** Natural limit of 10 iterations from `orchestrateLlmLoop`
- **Data freshness:** Data loaded from IndexedDB on each call (no stale cache)
- **Deleted data:** `isDeleted` filter prevents searching deleted topics/messages
